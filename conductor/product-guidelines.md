# Product Guidelines

## Tone & Voice
**Conversational & Approachable:** All documentation, guides, and playbooks should be written in a friendly, encouraging, and easy-to-read manner. The goal is to make complex AI concepts accessible, especially for beginners.

## Documentation Style
**Playbook-Driven:** We prioritize practical, step-by-step guides and how-tos. Developers should be able to follow a clear path from setup to execution rather than getting bogged down in exhaustive API references right away.

## Developer Experience (DX)
**Speed First:** The framework is optimized for the absolute fastest time from zero to a running prototype. We want developers to experience the magic of building AI agents as quickly as possible without unnecessary friction.

## AI Integration Philosophy
**AI as an Assistant:** The framework is built around the concept of "Agent-Driven Development." The human developer provides the overarching direction and architectural guardrails, while the AI assistant safely executes the coding tasks within those defined boundaries.
